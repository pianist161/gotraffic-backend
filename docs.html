System Overview

The BiTrans → SEPAC Migration Tool automates the conversion of traffic signal timing data from BiTrans controller format to SEPAC controller format. It replaces the manual VBA-based workflow (MDC Data Prep v4.4) with a modern web application supporting batch processing, validation, and export.

Conversion Algorithm

The conversion engine replicates the VBA macros from the MDC Data Prep v4.4 workbook. It transforms BiTrans cumulative ForceOff points into individual SEPAC phase splits within a standard NEMA dual-ring, 8-phase structure.

SEPAC Split Formulas

Where FOn = ForceOff for phase n, CL = Cycle Length, Sn = computed SEPAC split for phase n.

// Leading phases (direct or delta)
Ph3 = IF(FO3=0, 0, FO3)
Ph4 = IF(FO4=0, 0, IF(FO3=0, FO4, FO4 - FO3))
Ph7 = IF(FO7=0, 0, FO7)
Ph8 = IF(FO8=0, 0, IF(FO7=0, FO8, FO8 - FO7))

// Cross-ring phases (reference barrier partner)
Ph1 = IF(FO1=0, 0,
       IF(FO3+FO4=0,
          IF(FO8=0, FO1-FO7, FO1-FO8),
          IF(FO4=0, FO1-FO3, FO1-FO4)))

Ph5 = IF(FO5=0, 0,
       IF(FO7+FO8=0,
          IF(FO4=0, FO5-FO3, FO5-FO4),
          IF(FO8=0, FO5-FO7, FO5-FO8)))

// Through phases (remainder of cycle)
Ph2 = IF(CL=0, 0,
       IF(FO3+FO4=0,
          IF(FO1=0, CL-S7-S8, CL-FO1),
          IF(FO1=0, CL-S3-S4, CL-FO1)))

Ph6 = IF(CL=0, 0,
       IF(FO7+FO8=0,
          IF(FO5=0, CL-S3-S4, CL-FO5),
          IF(FO5=0, CL-S7-S8, CL-FO5)))

Minimum Split Validation

Without Pedestrians

MinSplit = MinGreen + YellowChange + RedClear + 1

With Pedestrians

MinSplit = MAX(MinGreen, Walk + FDW) + YellowChange + RedClear + 1

Validation Result

Each phase in each plan receives one of three statuses:

  • PASS — SEPAC split ≥ minimum split
  • FAIL — SEPAC split > 0 but < minimum split (highlighted red in export)
  • INACTIVE — both SEPAC split and minimum split are 0

FDOT Override Logic

On every upload, after BiTrans parsing and before conversion, the system automatically applies FDOT-required timing values:

  1. Looks up the asset in the master_intersections table (via phases_json)
  2. For each phase (1-8) and each bank (1-3), replaces yellow_change, red_clear, and ped_fdw with FDOT values
  3. Every override is logged to the fdot_overrides table (original vs. FDOT value)
  4. SEPAC splits are then computed using the corrected timing values

Security & CORS

CORS Configuration

The API uses a custom DynamicCORSMiddleware that validates request origins against:

  • Explicit origins from the CORS_ORIGINS environment variable (comma-separated)
  • Dynamic Vercel pattern — any https://*.vercel.app subdomain is automatically allowed (supports preview deployments)

Production origins include:

https://gotrafficusa.com
https://www.gotrafficusa.com
http://localhost:3000  (development)

Environment-Based Configuration

All sensitive settings are loaded from environment variables via pydantic-settings:

  • DATABASE_URL — PostgreSQL connection string (never committed to code)
  • CORS_ORIGINS — allowed origins list
  • PORT — server port (Railway sets this automatically)

File Upload Security

  • File type validation — only .xls, .xlsx, and .xlsm accepted
  • Files stored in a server-side uploads/ directory, not publicly accessible
  • Filenames are preserved for reference but not used in file paths for execution

Getting Started

Open the application in your browser. You will see the Dashboard with:

  • Migration Statistics — counts of intersections by status (pending, converted, exported, complete)
  • Intersection List — searchable, filterable table of all uploaded intersections
  • Upload Area — buttons to upload master list and BiTrans files
  • Navigation — sidebar or top nav to access all sections

The typical workflow is: Upload Master List (once) → Upload BiTrans files → Review & Validate → Export.

1

Upload Master List

The FDOT Master List is an Excel workbook (.xlsm or .xlsx) containing FDOT-mandated yellow change, red clearance, and pedestrian clearance times for every intersection in the county.

Where to find it

The master list is maintained by the Traffic Signals Division. The file is typically named similar to MDC Data Prep - Ver 4.4.xlsm.

How to upload

  1. Click "Upload Master List" on the dashboard
  2. Select the master list file from your computer
  3. The system will parse all intersection entries and store them
  4. You will see a confirmation with the number of entries loaded

Important notes

  • Uploading a new master list replaces all existing entries
  • This only needs to be done once, or when FDOT timing values are updated
  • The master list must be uploaded before BiTrans files for FDOT overrides to take effect
2

Upload BiTrans XLS

Each BiTrans export file (.xls) contains 9 sheets representing one intersection’s complete signal timing data.

How to upload

  1. Click "Upload BiTrans File"
  2. Select a single .xls file exported from BiTrans
  3. The system will:
    • Parse all 8 pages of timing data
    • Apply FDOT overrides from the master list (if uploaded)
    • Automatically run the ForceOff → SEPAC conversion
    • Create a migration record with status "converted"
  4. On success, you will see the intersection’s asset number, location, and counts of plans, detectors, and overlaps

Expected result

The intersection appears in the intersection list, and you can click on it to view full details.

3

View Intersection

Click on any intersection in the list to open its detail page.

Available tabs

  • Overview — Metadata, phase movements, zone assignments, equipment info
  • Phase Timings — Walk, FDW, min green, extension, max limits, yellow, red clear for all 8 phases across 3 banks
  • Coordination Plans — Plans 1-30 with cycle length, offset, ForceOff values, and computed SEPAC splits
  • TOD Schedule — Time-of-day plan switching events and holiday events
  • Detectors & Overlaps — Detector assignments, delays, extensions; overlap parent phases and clearance times
  • Preemption — Preemption configurations (if applicable)
  • FDOT Diff — Side-by-side comparison of original BiTrans values vs. FDOT overrides applied
  • Comments — Notes and observations attached to the intersection
4

Run Conversion

Conversion runs automatically during upload. To re-run conversion (e.g., after master list update):

  1. Navigate to the intersection detail page
  2. Click "Run Conversion"
  3. The system recalculates SEPAC splits for all active plans (cycle length > 0)
  4. Results appear immediately in the Coordination Plans tab

What to check

  • Verify that active plans show SEPAC split values in the split columns
  • Confirm the migration status is updated to "converted"
  • Check for any warning messages about unusual ForceOff patterns
5

Validate Results

Validation compares each SEPAC split against the minimum split requirement for that phase.

Understanding the results

  • PASS (green) — The computed split meets or exceeds the minimum. No action needed.
  • FAIL (red) — The split is below the minimum. Manual review required. You may need to override the split value or adjust the coordination plan.
  • INACTIVE (gray) — The phase is not used in this plan. No action needed.

Minimum split components

The minimum split view breaks down the calculation showing: MinGreen, Walk, FDW, YellowChange, RedClear, and the +1 second startup time. This helps identify which component is driving a failure.

6

Override Splits

When a SEPAC split fails validation, or when engineering judgment requires a different value:

  1. Navigate to the intersection’s conversion results
  2. Click "Override" on the phase/plan you want to modify
  3. Enter the new split value and a reason for the change
  4. Click "Save"

Override tracking

  • Every override is logged with the original value, new value, reason, and timestamp
  • View the override history via the "Overrides" tab
  • To undo overrides: click "Reset Splits" to revert to auto-computed values (for one plan or all plans)
7

Export

Once conversion and validation are complete, export the results in the format you need.

Export formats

  • Excel (.xlsx) — Matches the VBA "Calculated Split Times" layout with:
    • Three plan groups (Plans 1-10, 11-20, 21-30)
    • Minimum splits tables (with and without PEDs)
    • Phase timing summary sheet
    • Failing splits highlighted in red
  • PDF — Printable conversion report with:
    • Intersection header info
    • Phase timing table
    • ForceOff vs. SEPAC splits per plan with pass/fail indicators
    • FDOT overrides table
  • JSON — Machine-readable export of full intersection data (timings + plans)

How to export

  1. From the intersection detail page, click the "Export" dropdown
  2. Select your preferred format
  3. The file will download automatically
  4. Migration status is updated to "exported"

Batch Operations

Batch Upload

Upload multiple BiTrans XLS files at once:

  1. Click "Batch Upload"
  2. Select multiple .xls files (hold Ctrl/Cmd to select multiple)
  3. The system processes each file individually and reports per-file results
  4. Files that fail parsing show an error message without affecting other files

The response includes a summary: total files processed, successes, and failures.

Batch Excel Export

Export multiple intersections into a single Excel workbook:

  1. Select the intersections you want to export (by checking boxes)
  2. Click "Batch Export"
  3. The system generates a multi-sheet workbook with one sheet per intersection

Troubleshooting

"File format not supported" when uploading
The system only accepts .xls, .xlsx, and .xlsm files. Make sure your BiTrans export is saved in the correct format. Some BiTrans versions export as .xls (Excel 97-2003). The master list can be .xlsm or .xlsx.
FDOT overrides not applying
Make sure the master list is uploaded before uploading BiTrans files. If you uploaded BiTrans files first, upload the master list, then re-upload the BiTrans files or use "Run Conversion" to re-process with FDOT values. Also verify that the asset numbers in the master list match those in the BiTrans files.
All SEPAC splits are 0
This typically means the coordination plan has a cycle length of 0 (inactive plan). Only plans with cycle length > 0 are converted. Check the Coordination Plans tab to verify cycle lengths.
Phase shows FAIL but the split looks correct
Check the minimum split breakdown to see which component is causing the failure. Common causes: a pedestrian FDW value (from FDOT override) that pushes the minimum above the computed split. You can manually override the split if engineering judgment supports it.
Upload succeeds but some data is missing
Check the warnings returned in the upload response. The parser tolerates missing sheets or empty cells, but logs warnings for anything unexpected. Verify the XLS file was exported completely from BiTrans (should contain 9 sheets labeled "Page 1 of 8" through "Page 8 of 8" plus "Unitized").
Cannot connect to the application
Verify you are using the correct URL. The frontend is hosted on Vercel and the API on Railway. Check that you have internet access. If the API is down, the frontend will show a connection error banner. Contact IT support if the issue persists.
Export does not include all plans
Only active plans (cycle length > 0) appear in the export. The Excel export groups plans into three sheets: Plans 1-10, 11-20, and 21-30. Inactive plans are intentionally omitted.
Batch upload partially failed
The batch upload response lists each file with its result. Files that fail do not affect other files. Common causes: duplicate asset number (already uploaded), corrupt file, or unexpected sheet layout. Fix the individual files and re-upload them.
